Profiles

The following profiles have been defined for this implementation guide. They are organized by module to support the MII core dataset requirements.

Person Module Profiles

  • MII PR Person Patient - Patient demographics and administrative information
  • MII PR Person Patient Pseudonymisiert - Pseudonymized patient for research use
  • MII PR Person Vitalstatus - Vital status observation
  • MII PR Person Todesursache - Cause of death observation

Treatment Case Module Profiles

  • ​MII PR Fall Kontakt Gesundheitseinrichtung - Encounter/contact with healthcare facility

Diagnosis Module Profiles

  • MII PR Diagnose Condition - Diagnosis/condition

Procedure Module Profiles

  • MII PR Prozedur Procedure - Procedure/intervention

Extensions

The following extensions have been defined as part of the MII Implementation Guide. A registry of standard extensions can be found in the FHIR specification and additional extensions may be registered on the HL7 FHIR-Registry.

Treatment Case Module Extensions

  • MII EX Fall Planned Start Date - Planned start date for encounter
  • MII EX Fall Planned End Date - Planned end date for encounter

Procedure Module Extensions

  • MII EX Prozedur Durchfuehrungsabsicht - Procedure intent
